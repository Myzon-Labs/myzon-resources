## ğŸ“ Naming Conventions

Use **lowercase letters and hyphens** for all filenames, including the plugin/mod name:

```
plugin-name-main-interface.png
plugin-name-settings-page.jpg
feature-demo-01.png
tutorial-step-01-setup.png
comparison-before-after.png
```

**Format:** `[project-name]-[feature/section]-[description].[extension]`

- `project-name`: lowercase, hyphens between words
- `feature/section`: what the screenshot shows (e.g., main-interface, settings, config)
- `description`: optional, additional context (e.g., 01, 02 for sequences)
- `extension`: file format

## ğŸ“¤ Upload Rules

1. **Use PNG for best quality** (lossless compression)
2. **Maximum file size:** 2-5 MB per image (optimize before upload)
3. **Minimum resolution:** 1024x768px for full-screen; maintain aspect ratio
4. **Recommended resolution:** 1280x720px or 1920x1080px for standard screens
5. **Include context:**
   - Add descriptive filenames with version/date if needed
   - Update this README with brief descriptions
6. **Organize by plugin/mod:**
   - `screenshots/plugin-name/` or prefix filenames with plugin name
7. **Screen capture best practices:**
   - Remove personal information and sensitive data
   - Use consistent window sizes
   - Clean up desktop/UI before capturing
   - Use high DPI for clarity
8. **For comparison images:**
   - Keep before/after screenshots consistent (same resolution, similar crops)
   - Clearly label or separate by naming

## ğŸ¨ Design Guidelines

- **Consistent framing:** Use same window/screen size for series
- **Clarity:** Ensure text is readable (at least 12pt font visible)
- **Highlighting:** Use annotations or arrows to highlight key features
- **Consistency:** Match theme/branding across all screenshots
- **Cleanliness:** Remove clutter, personal data, or test content
- **Time stamps:** Include version/date if relevant (in filename or overlay)
